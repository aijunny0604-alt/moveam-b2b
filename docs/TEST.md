## 핵심 시나리오

### 보안 (최우선)

- [ ] 비로그인 상태에서 제품 API 직접 호출 → 데이터 0건 (RLS 확인)
- [ ] 차단(is_active=false)된 업체 계정 로그인 → 열람 불가
- [ ] vendor 계정으로 제품 수정 시도 → 거부
- [ ] Storage 이미지 URL 비로그인 접근 → 거부 (버킷 정책)

### 업체 열람 흐름 (폰 기준)

- [ ] 로그인 → 브랜드 → 제품 목록 → 상세 → 뒤로가기 정상
- [ ] 검색: "타각킷", "젠쿱", "허브" 등 통용어로 검색됨
- [ ] 옵션 여러 개 제품(허브스페이스 등) 단가표 전부 표시
- [ ] 카톡/전화 버튼 → 실제 앱 연결
- [ ] 하단 버튼이 모바일 브라우저 UI에 가리지 않음

### 관리자 흐름

- [ ] 제품 등록 + 사진 업로드 → 목록 즉시 반영
- [ ] 단가/재고(품절)/주의사항 수정 → 업체 화면 반영
- [ ] 업체 코드 발급 → 해당 코드로 입장 가능 / 코드 변경 → 구 코드 무효
- [ ] 차단 → 해당 업체 즉시 열람 불가

### 주문 흐름 (v0.4, 2026-07-06 라이브 검증 완료)

- [ ] 업체: 옵션 선택 → 견적함 담기 → 수량 조절 → 주문 신청 → orders 기록
- [ ] 관리자: 새 주문 토스트 + 관리자 버튼 뱃지 → 주문 탭에서 수락/거절(+메모)
- [ ] 업체: 상태 변경 토스트 + 주문 내역 실시간 갱신 (새로고침 없이)
- [ ] 발주서 복사 → 카톡 붙여넣기 형식 확인
- [ ] RLS: 업체 A가 업체 B 주문 조회 불가
