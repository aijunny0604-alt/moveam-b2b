# AUTH

## Purpose

업체별 로그인 계정과 관리자 권한 규칙. 도매 단가 접근 통제가 이 서비스의 핵심 보안 요구사항이다.

## Current State

설계안 단계.

## 계정 모델

| 역할 | 할 수 있는 것 |
|------|---------------|
| vendor (업체) | 로그인, 제품/단가/사진 열람, 검색, 문의 버튼 |
| admin (우리) | vendor 전부 + 제품/단가/사진 관리, 업체 계정 발급·차단 |

- 회원가입 없음. **계정은 전부 관리자가 발급**한다 (Supabase Auth admin API 또는 대시보드).
- 사장님들이 이메일이 없을 수 있으므로 아이디는 `업체명@moveam.local` 형태의 가짜 이메일 + 초기 비밀번호로 발급하고, 로그인 화면 라벨은 "아이디/비밀번호"로 표기.
- 거래 중단 업체는 삭제 대신 `profiles.is_active = false`로 차단 (RLS에서 즉시 열람 불가).

## 로그인 흐름

1. 미로그인 접속 → 로그인 화면만 표시 (제품 데이터 요청 자체가 401/RLS 차단)
2. 로그인 성공 → 브랜드/제품 목록
3. `role = 'admin'`이면 상단에 관리자 메뉴 노출

## Current Rules

- 도매가는 서버(RLS)에서 차단하는 것이 원칙. 프론트 숨김은 UX일 뿐 보안이 아니다.
- 비밀번호 재설정은 관리자가 재발급하는 방식으로 단순화 (이메일 발송 플로우 없음).
- 세션은 Supabase 기본 (localStorage persist). 공용 PC 고려사항 생기면 재검토.

## 확장 여지 (지금은 안 함)

- 업체 등급별 단가 차등 (profiles에 tier 추가 + 가격 테이블 분기)
- 열람 로그 (누가 언제 어떤 제품을 봤는지)
