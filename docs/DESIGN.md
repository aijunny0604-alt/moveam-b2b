# DESIGN

## Purpose

화면 구성과 반복 컴포넌트 규칙. 모바일 우선 — 사장님들은 대부분 폰으로 본다.

## Current State

설계안 단계. 기존 디자인 없음 — 깔끔한 B2B 카탈로그 스타일로 새로 만든다.

## 화면 목록

| 화면 | 경로 | 내용 |
|------|------|------|
| 로그인 | (비로그인 시) | 업체: 코드 1칸 / 관리자: 아이디+비번 토글 |
| 메인 (전체 브라우저) | `/` | 전체 검색 + 브랜드 칩 필터 + 제품 그리드 (2~3열 반응형) |

| 견적함 | `/cart` | 수량 조절, 요청사항, 주문 신청, 발주서 카톡 복사 |

## 핵심 컴포넌트 (반복 사용)

- **ProductCard** — 사진 + 제품명 + 도매/소매가 + 뱃지(품절/옵션 N개/주의사항). fade-up 스태거, 호버 리프트
- **PriceTable** — 스토어형 옵션 선택: 칩 → 선택 옵션 큰 가격 카드 → 접이식 전체 단가표(8개 초과 시 기본 접힘) + 견적함 담기 버튼
- **SearchBar** — 제품명+search_keywords 즉시 필터 (클라이언트, 수백 건 규모라 충분)
- **InquiryButtons** — 카톡/전화 하단 고정 (config.js 값 없으면 숨김)
- **Header** — 로고, 관리자(주문 대기 뱃지), 견적함(수량 뱃지), 로그아웃

## 모션 규칙 (index.css)

- `.fade-up` 카드 스태거(최대 12개까지 35ms 간격), `.fade-in` 이미지 전환, `.skeleton` 로딩 시머
- 절제 원칙: 정보 전달 우선, 튀는 모션 금지. `prefers-reduced-motion` 존중
- 해상도 스케일: 1280px↑ 17px, 1920px↑ 19px root font

## 스타일 규칙

- Tailwind CSS v3. 다크 배경보다 밝은 카탈로그 톤 (사진이 주인공)
- 가격 표기: `770,000원` 천단위 콤마, 도매가는 강조색/소매가는 보조색으로 구분
- 터치 영역 최소 44px (기존 flow-check 피드백 반영)
- 하단 고정 버튼은 모바일 네비/브라우저 UI와 겹치지 않게 `safe-area` + 여백 확인
- 애니메이션 최소 — 정보 전달 우선. 로딩은 스켈레톤 카드 정도만

## Current Rules

- 컴포넌트는 `src/components/`, 페이지는 `src/pages/` — 파일 분리 유지
- 이미지: 업로드 시 리사이즈(긴 변 1200px)해서 Storage 용량/로딩 관리. 카드용 썸네일은 CSS로 처리(초기), 트래픽 커지면 썸네일 변환 도입
