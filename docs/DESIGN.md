# DESIGN

## Purpose

화면 구성과 반복 컴포넌트 규칙. 모바일 우선 — 사장님들은 대부분 폰으로 본다.

## Current State

설계안 단계. 기존 디자인 없음 — 깔끔한 B2B 카탈로그 스타일로 새로 만든다.

## 화면 목록

| 화면 | 경로 | 내용 |
|------|------|------|
| 로그인 | `/login` | 아이디/비밀번호 |
| 브랜드 홈 | `/` | 브랜드 카드 목록 (번웨이, HKS…) |
| 제품 목록 | `/brand/:slug` | 검색바 + 제품 카드 그리드 |
| 제품 상세 | `/product/:id` | 사진 슬라이드, 도매/소매가, 옵션별 단가표, 문의 버튼 |
| 관리자 | `/admin` | 제품 CRUD, 사진 업로드, 업체 계정 관리 |

## 핵심 컴포넌트 (반복 사용)

- **ProductCard** — 대표 사진 + 제품명 + 도매/소매가. 목록·검색결과 공용
- **PriceTable** — 옵션(variants)별 단가 표. 상세·관리자 미리보기 공용
- **SearchBar** — 제품명 + search_keywords 대상 즉시 필터 (클라이언트 필터, 데이터 수백 건 규모라 충분)
- **InquiryButtons** — 카톡 채널 + 전화 연결 고정 버튼 (제품 상세 하단 sticky)
- **AdminForm** — 제품 등록/수정 폼 + 이미지 업로더

## 스타일 규칙

- Tailwind CSS v3. 다크 배경보다 밝은 카탈로그 톤 (사진이 주인공)
- 가격 표기: `770,000원` 천단위 콤마, 도매가는 강조색/소매가는 보조색으로 구분
- 터치 영역 최소 44px (기존 flow-check 피드백 반영)
- 하단 고정 버튼은 모바일 네비/브라우저 UI와 겹치지 않게 `safe-area` + 여백 확인
- 애니메이션 최소 — 정보 전달 우선. 로딩은 스켈레톤 카드 정도만

## Current Rules

- 컴포넌트는 `src/components/`, 페이지는 `src/pages/` — 파일 분리 유지
- 이미지: 업로드 시 리사이즈(긴 변 1200px)해서 Storage 용량/로딩 관리. 카드용 썸네일은 CSS로 처리(초기), 트래픽 커지면 썸네일 변환 도입
